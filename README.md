# Elite Clock ⭐

A Sinclair Spectrum/BBC Micro **Elite** ship clock for the **ESP32-C3 Super Mini** with a 2.8" UC8230 (ILI9341-compatible) 240×320 colour LCD.

Ships rotate in 3-D on a star-field background with an authentic Elite-style radar panel, clock, and ship stats — all configurable via a green-on-black web UI and Home Assistant.

---

## Features

| Feature | Detail |
|---|---|
| **14 classic Elite ships** | Cobra Mk III · Viper · Sidewinder · Mamba · Krait · Python · Anaconda · Boa · Asp Mk II · Fer-de-Lance · Moray · Thargoid · Missile · Worm |
| **3-D rendering** | Wireframe (authentic Elite look) or flat-shaded with diffuse lighting & painter's sort |
| **Back-face culling** | Only visible faces drawn — correct Elite algorithm |
| **Auto ship cycling** | Configurable 10 s – 10 min |
| **Touchscreen zones** | Prev · Next · Shaded toggle · Radar info toggle |
| **Web control panel** | Green-on-black Elite-themed UI at local IP |
| **Home Assistant** | MQTT auto-discovery — ship select, shaded switch, speed/duration sliders, prev/next buttons |
| **Wi-Fi auto-connect** | Saves credentials; launches setup AP if no network found |
| **Manual time** | Set time without NTP (for offline use) |
| **GMT/DST** | Any timezone offset −12 … +14 with optional +1 h DST |
| **NVS persistence** | All settings survive power loss |

## Hardware

### Components

| Component | Details |
|---|---|
| MCU | ESP32-C3 Super Mini (160 MHz, 4 MB flash, USB-C) |
| Display | 2.8" SPI TFT 240×320 with UC8230 driver |
| Touch | XPT2046 resistive touch (usually integrated on display module) |

### Wiring

```
ESP32-C3          2.8" TFT module
──────────────────────────────────────────
GPIO 6  (MOSI) ──→ SDI / MOSI
GPIO 4  (SCK)  ──→ SCK / CLK
GPIO 7         ──→ CS  (display chip select)
GPIO 2         ──→ DC  / RS (data/command)
GPIO 3         ──→ RST
GPIO 8         ──→ BL  / LED (backlight, 3.3 V)
GPIO 9         ──→ T_CS (touch chip select)
GPIO 10        ──→ T_IRQ (touch interrupt)
GPIO 5  (MISO) ──→ SDO / MISO (touch data back)
3.3 V          ──→ VCC
GND            ──→ GND
```

> **Note:** MISO (GPIO 5) is shared between display and touch on most modules.

## Build & Flash

### Using PlatformIO (recommended)

```bash
# Install PlatformIO CLI
pip install platformio

# Clone and enter the project
git clone https://github.com/YOUR_USERNAME/elite-clock.git
cd elite-clock/elite-clock

# Build firmware
pio run

# Build LittleFS image (web UI)
pio run --target buildfs

# Flash everything (replace PORT)
pio run --target upload                    # firmware
pio run --target uploadfs                  # filesystem
```

### Using GitHub Actions

Push to `main` → workflow builds automatically. Download the artefact zip from the **Actions** tab.

Tag a release to also create a GitHub Release with all binary files attached:
```bash
git tag v1.0.0
git push origin v1.0.0
```

### Using esptool.py directly

See `artifacts/FLASH_INSTRUCTIONS.md` generated by the workflow.

## First Boot

1. Power on — you will see the boot splash.
2. **No Wi-Fi saved:** The device launches a hotspot:
   - SSID: `EliteClock-Setup`
   - Password: `eliteclock`
   - Browse to **http://192.168.4.1**
3. Enter your Wi-Fi credentials and save.
4. Device reboots and connects. Find the IP in your router's DHCP table.
5. Browse to the device IP for the full control panel.

## Web Control Panel

Navigate to the device IP in a browser. Controls:

- **Previous / Next ship** buttons
- **Wireframe / Shaded** toggle
- **Rotation speed** slider
- **Ship duration** slider (10 s – 10 min)
- **Wi-Fi setup** (SSID scan + password)
- **Time setup** (GMT offset, DST, manual time entry)

## Home Assistant

The device publishes MQTT auto-discovery payloads so it appears in HA automatically. Entities created:

| Entity | Type | Description |
|---|---|---|
| Ship Selection | Select | Choose which ship to display |
| Shaded Mode | Switch | Wireframe ↔ Shaded toggle |
| Rotation Speed | Number | 0.1 – 5.0 °/frame |
| Ship Duration | Number | 10 – 600 seconds |
| Previous Ship | Button | Cycle backwards |
| Next Ship | Button | Cycle forwards |

Configure the MQTT broker in the web UI or HA tab.

## Architecture

```
src/
├── main.cpp       ← Application loop, display, touch zones
├── ships.h        ← 14 ship vertex/edge/face definitions (Elite geometry)
├── renderer.h     ← 3-D pipeline: rotate → project → cull → wireframe/shade
├── settings.h     ← NVS Preferences wrapper (all config)
├── wifi_mgr.h     ← Auto-connect STA / AP fallback / NTP sync
├── web_ui.h       ← AsyncWebServer REST API (/api/*)
└── ha_mqtt.h      ← PubSubClient + HA MQTT auto-discovery

data/
└── index.html     ← Elite-themed single-page web UI (served from LittleFS)
```

## Customisation

### Adding a ship

In `ships.h`:
1. Define `myship_v[]` (vertices), `myship_e[]` (edges), `myship_f[]` (faces).
2. Add an entry to the `SHIPS[]` array.
3. Bump `NUM_SHIPS`.

Vertex coordinates are `int8_t` (−127 … +127). The renderer auto-scales to fit the viewport.

### Touch calibration

If touches are off, adjust the constants in `main.cpp`:
```cpp
#define TOUCH_XMIN  300
#define TOUCH_XMAX 3800
#define TOUCH_YMIN  300
#define TOUCH_YMAX 3800
```

Use the serial monitor at 115200 baud to print raw `p.x / p.y` values.
